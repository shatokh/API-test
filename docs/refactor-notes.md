# Refactor notes & improvement opportunities

## Точки для улучшения

- **Конфигурация окружения**: заменить прямой `process.exit` на централизованный обработчик ошибок/валидатор env (например, `envalid` или `zod`), чтобы видеть все проблемы сразу, а не по одной.
- **Логирование**: добавить структурированные логи (JSON) и уровни, чтобы легче было отправлять логи в ELK/Datadog.
- **MongoDB lifecycle**: вынести подключение/отключение в отдельный модуль и добавить обработчики `SIGTERM`, чтобы корректно завершать контейнеры в Kubernetes.
- **Healthcheck**: расширить `/health` до `/ready` и `/live`, чтобы различать доступность сервиса и готовность к приёму трафика.
- **Swagger**: вынести `swaggerUiOptions` в отдельный конфиг и добавить версионирование API в документации.
- **Observability**: добавить `request-id` middleware и прокидывать его в логи для трассировки запросов.
